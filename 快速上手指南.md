> **1 分钟** （只需要4条命令）从零装好 ToPGauge，*无需配置* \*，让 Claude Code 状态栏显示你的外接Provider的 **token 套餐用量 / 剩余额度**（或 **账户余额**）。想要每个字段、每个模块的完整说明，请看 [MANUAL.md](./MANUAL.md)。

- 如果你正在使用 `claude-hud` 或 `ccstatusline`，ToPGauge 会在它们的状态栏下方渲染自己的行，不破坏原有布局。/topgauge:install 和 /topgauge:uninstall 会正确处理共存方案。
- ToPGauge 绝不会收集你的 token，也不会把它们写入日志或落盘。运行过程中，ToPGauge 只会把 token 作为 Bearer 头传给 provider 的套餐端点。详情请看 [SECURITY.md](./SECURITY.md)。

## 1. 简介

ToPGauge 是一个 **provider 无关** 的 Claude Code 状态栏插件，专注于外接 provider 特有的 **quota / balance** 数据，外加从 Claude Code stdin 读取的轻量用量统计。

> Claude 套餐的额度信息通常包含在 stdin 数据中，没订阅过，因此无法确认验证。现有 `claude-hud` / `ccstatusline` 已经可以显示。不过它们是 **内置 provider 绑定** 的，ToPGauge 则更多考虑 **外接 provider** 的额度实时显示，避免在心流工作状态中可能受到 provider 额度限制而被迫中断。当然，ToPGauge 也会在合适的时机加入对 Claude 套餐的额度信息显示的支持。

它按 `ANTHROPIC_BASE_URL` (settings.json) 决定渲染什么，因此同一个插件对任何受支持 provider 的套餐端点都能用，**无需按 provider 重装，也无需繁琐配置**。当前内置支持：

| `ANTHROPIC_BASE_URL`                 | 渲染行                   | 数据来源                         |
| ------------------------------------ | ------------------------ | -------------------------------- |
| `https://api.minimaxi.com/anthropic` | `Usage: …` / `Remain: …` | `/v1/token_plan/remains`（MiniMax） |
| `https://api.deepseek.com/anthropic` | `Balance: …`             | `/user/balance`（DeepSeek）      |
| 其他（原生 Anthropic、OpenRouter…） | （隐藏）                 | —                                |

对未识别的 provider，插件会**隐藏 Quota / Balance 行**，并原样透传上游状态栏（如 `claude-hud` / `ccstatusline`）。 

ToPGauge 设计了可扩展性极好的**插件的插件** （the Plugins of Plugin, PoP）架构，通过动态导入的方式实现更多 provider 的支持，只需要一个 *.js 文件和一段 provider 节点配置，便可实现一个新 provider 的支持。如果你愿意，可以创建一个自定义插件（ESM 文件）来接入其他 provider 的套餐端点，**无需 fork**。完整 ABI、填充契约、覆盖解析顺序、内置插件和仓库现有的外部插件目录（包括 kimi 的凭证获取、copilot-api 的 localhost 端点），以及用 provider 参数设定显示条件的方法，都在 [HOW_TO_CREATE_A_PLUGIN.md](./HOW_TO_CREATE_A_PLUGIN.md)。也欢迎分享其他 providers 的插件，或者提 PR 把它们放进内置目录。


## 2. 安装

**第一步 —— 装插件**（这是一个单插件市场）：

```
/plugin marketplace add cwf818/topgauge
/plugin install topgauge@topgauge
/reload-plugins
```

> `/reload-plugins` 必不可少：让加载器识别新命令。忘了这步是安装后「command not found」的最常见原因。

**第二步 —— 接入 `settings.json`**：

```
/topgauge:install
```

这会给当前 `settings.json` 打补丁（默认用户级；加 `--project` 走项目级）。若已被本插件接管（`_topgauge_managed: true`），则空操作。

**卸载**（自包含清理，即使缓存已被清也能用；不动 `env` 里的 token；默认会保留插件数据，加 `--completely` 参数完全卸载，包括配置文件、自定义插件目录和缓存目录）：

```
/topgauge:uninstall
```

**其他命令**：`/topgauge:clean`（清理旧 `.bak.<ts>` 备份）、`/topgauge:clean-cache`（删除旧版本缓存目录，只留最新）。

## 3. 基本配置

配置文件路径（启动时加载一次）：

- **Unix**：`~/.claude/plugins/topgauge/config.json`
- **Windows**：`%USERPROFILE%\.claude\plugins\topgauge\config.json`

一份含全部字段的参考：[config.example.json](./config.example.json)——复制到上面的路径后编辑即可。

### 最常改的几项

**显示模式**——默认 `remaining`（行首 `Remain:`，显示剩余百分比）。切成已用模式：

```json
{ "display": "used" }
```

`used` 模式行首变 `Usage:`，显示已用百分比，进度条彩色段代表已用部分。（`display` 仅对 Quota 有效；DeepSeek 的 `Balance:` 没有百分比可翻转。）

**配色阈值**——5 段配色的分界：

```json
{
  "thresholds": {
    "percentBands": [60, 70, 80, 90],
    "balanceBands": [5, 10, 20, 50]
  }
}
```

**刷新缓存 TTL**——一次成功 API 响应缓存多久（默认 60 秒）：

```json
{ "cacheTtlMs": 60000 }
```

**布局模板**——用 `statuslineTemplate` + `lineTemplates` 拼你想要的行，`lineTemplates` 用于使用内置可视化模块配置基础模板，`statuslineTemplate` 用于使用内置可视化模块和 `lineTemplates` 配置状态栏的呈现模板。

`statuslineTemplate` 提供了4中快速的预设：`simple`、`compact`、`standard`、`abundant`，可以直接在config.json中设置使用。

`lineTemplates` 提供了 `quota`、 `balance`、 `information`、 `mem_info`、 `git_info`、 `git_info_all`、 `tick_eval` 等预设内置模板，用户可以通过 `m_template` 模块使用，也可以自定义更多模板。

ToPGauge 提供了配置非常灵活的配置方法，模板、模块目录与 inline-args 语法见 [MANUAL.md §2](./MANUAL.md#2-top-level-schema) 与 [§8 模块参考](./MANUAL.md#8-module-reference)。

## 4. 插件（接入自定义 provider）

内置只带两个 provider（`minimax` / `deepseek`）。要接入 **kimi、moonshot、z.ai、私有网关** 等其他 provider，**无需 fork**——往指定目录 （`~/.claude/plugins/topgauge/query_plugins/<id>`） 丢一个 ESM 文件，并配置 provider 节点即可静默覆盖内置链路。

最短路径（三步）：

```bash
# 1. 建插件目录（跨缓存滚动稳定）
mkdir -p ~/.claude/plugins/topgauge/query_plugins/<id>

# 2. 放插件文件，导出 default { fetchAccountCredit(authenticationKey, ctx) }
#    返回 Partial<Quota> 或 Partial<Balance>，host 会补全规范形状

# 3. 在 config.json 的 providers.<id> 里登记 BASE_URL_COMPARED_TO 等字段
```

其中 `<id>` 必须与 `config.json` 里 `providers.<id>` 的键一致。完整 ABI、填充契约、覆盖解析顺序、内置插件目录（kimi 的凭证获取、copilot-api 的 localhost 端点），以及用 provider 参数设定显示条件的方法，都在 [HOW_TO_CREATE_A_PLUGIN.md](./HOW_TO_CREATE_A_PLUGIN.md)。

## 5. 安全与协议

- 插件复用 `process.env.ANTHROPIC_AUTH_TOKEN` 调用 provider 端点，**不引入新环境变量**。
- token **仅**用作单次 GET/POST 的 `Bearer` 头，**从不**被日志记录、写入 stdout、出现在错误信息中或落盘。
- 卸载脚本本地运行、无 API 调用，**不触碰** `settings.json` 的 `env` 块（含你的 token）。
- 完整策略见 [SECURITY.md](./SECURITY.md)。发现漏洞请提 `security` 标签的 issue，**勿在报告中附真实 token**。
